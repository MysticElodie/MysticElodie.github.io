Morrowind Visual Masterguide

Introduction

While it's tempting to jump right in and start downloading mods, PLEASE read these introductory notes first, especially if you're new to Morrowind or Mod Organizer 2.

  • This guide assumes that you own a legal copy of Morrowind's basegame and its expansions. This means you should have either the official discs, or one of the many official digital releases.
    • Additionally, the guide is designed for players using the original engine. Some of the mods and programs used won't work correctly (or at all) with OpenMW.
  • It's always worth looking at a mod's description page (if it has one) before downloading it! Though this guide should cover everything you need to install each mod, it's best to know what you're installing.
  • The guide is designed with Mod Organizer 2 in mind. MO2 has official support for Morrowind, and in my opinion, it's the most convinient tool you can use for mod management. MO2 has a lot of different functions, and generally I'm not going to go too in-depth about how each one works. If you're coming to Morrowind from newer Bethesda games, there's a good chance you're already familiar with the program. If not, GamerPoets has a great set of videos on installing and using it, if you're confused about anything.
    • For many mods, MO2 will require you to point to a data directory folder when installing. Unless I specify otherwise in the guide, the data directory folder will always be the Data Files folder.
    • Some mods in this guide will need to be installed multiple times, or multiple files will need to be downloaded from a single mod page. When this happens, make sure to either merge or rename the mod unless you're fully reinstalling it.

Installation

  • You'll want to start with a new installation of the game. Preferably, you want to make sure that it's installed outside of the Program Files folder to prevent issues; if you're playing the Steam version, this page will show you how to change your install location for games, if you need to.
  • If you're playing any GOTY release of the game, then you already have the newest official patches. If you're playing on original disc releases, you can find the official patches for the game and its expansions towards the bottom of this page.
  • Before we do anything else, launch the game once to make sure everything is correctly installed. You don't have to go any further than the title screen.

External Tools

  • Morrowind Code Patch - This program patches many bugs in Morrowind's program. Manually download the main file, extract it into the game's main folder (The Morrowind folder, not Data Files) and run it.
    • MCP's most essential options are all checked by default, but I'd recommend looking through the other options too; a lot of them are very welcome changes. Some noteworthy examples;
      • Swift casting allows you to cast spells much more efficiently, and do so while holding a weapon.
      • Argonian clothing choice makes female Argonians use male meshes for armor and clothing, since they don't have defined breasts.
      • Hi-def cutscene support is required by some of the mods in this guide.
      • Map expansion (for Tamriel Rebuilt) is essential for TR users, zooming out the map so that all of the mainland is visible.
      • Shortcut key improvements offers many fixes and improvements for keyboard shortcuts.
      • Better typography improves the interfaces for books, scrolls, and the quest journal.
    • When you're done, click Apply chosen patches and then close the program.
  • MGE XE - This program's primary focus is visual improvements such as shaders and distant terrain. Manually download the MGE XE Manual Install file
    • Due to how MO2 works, we'll need to run MGE XE and several other programs through it. MO2 should automatically detect MGEXEgui.exe and add it to the executables list, but if not, you can add it yourself. Once it's added, launch the program via MO2.
    • We won't be changing most of MGE XE's settings until the end of the guide, but there's a few things we can do now.
      • On the Graphics tab, choose your preferred settings for resolution, windowed mode, antialiasing, etc. If "Auto FOV" isn't enabled by default, I recommend checking it. For now, don't enable shaders.
      • Skip over the Distant Land tab entirely for now, and go to the In-Game tab. By default, "Skip Opening Movie" is checked. If you want the Bethesda logo to display on startup, you can uncheck it. "Crosshair Autohide" is a nice change, hiding the crosshair when it isn't needed.
      • Still on the In-Game tab, change the Dynamic lighting coefficients to the following. This is the last thing we'll need to change in MGE XE for now;
        • Quadratic 2.619
        • Linear 1.0
        • Constant 0.382
    • Once you're done changing MGE XE's settings, you can close the program. Additionally, run the MWSE-Update.exe that came with it (you don't have to add it as an MO2 executable, just run it normally from the game's folder) to ensure you have the newest version of the Morrowind Script Extender installed.
  • mlox - mlox is Morrowind's equivalent to load order sorting tools such as LOOT. Download the mlox.exe file, and move it into your main game folder. Then, you'll need to add it to MO2.
    • After adding it, launch mlox from MO2 to make sure it launches properly. You shouldn't need to update your load order at this point, so you can just exit the program once it's finished loading.
    • After launching the program to setup its files, you'll want to update the base file. In most cases, this will be C:\Users\[Your Username]\AppData\Local\mlox\mlox. Once you find it, replace mlox_base.txt with the newest version.
  • Wrye Mash - Polemos Fork - Similar to Wrye Bash for newer Bethesda games, this tool provides a lot of useful functionality for modded installations of Morrowind. Download the Wrye Mash 2019 - x64 - manual installation archive file, and extract it into the game's main folder.
    • Before we can add Wrye Mash to MO2, we need to run it normally. Run mash64.exe in the Mopy folder, and click through the config wizard.
      • For the first entry, navigate to your main game folder.
      • The second entry's location doesn't matter; it's intended for anyone using Wrye Mash as a mod manager, which we aren't in this guide. (I just created a "Mods" folder within my main Morrowind folder for it).
      • For the Mlox directory option, select your Mlox.exe file.
    • You can keep the rest of the settings the same and finish setup, and then close Wrye Mash once it opens. Add Wrye Mash as an MO2 executable by adding the mash64.exe, and we're done with it for now.
  • tes3cmd - A program used for cleaning dirty mod files for Morrowind. Download the tes3cmd.exe file and place it into your Morrowind's Data Files folder. That's all we need to do with it for now.

Official Plugins

Bethesda developed several official plugins for Morrowind following its release. Information on these plugins can be found here. Unfortunately, many of these came with issues, so we won't be using the original versions.

  • Unofficial Morrowind Official Plugins Patched - This mod project includes the official plugins with many bugs fixed. You’re free to mix and match any of these; if there are any you don’t want to use, you can hide/delete their ESP files.
    • First, download and install the main UMOPP 3.0.3 file.
    • Next, download the Merged and Compatibility Versions file. What you install from here depends on which plugins you want to use.
      • If you plan on using all of the official plugins, only check UMOPP Better Armor Merged. After installing, hide/delete all of the .esp files from the UMOPP 3.0.3 main file.
      • If you only want to use some of the official plugins, only check UMOPP Compatibility. After installing, hide/delete all of the .esp files for plugins you don't want from both the main file and the compatibility versions file.
  • Pika's Miscellaneous Mods - Download the UMOPP Misc Mods and AddOns file. Enhancements for several of the official plugins and an improved script, respectively.
    • Install UMOPP Misc Mods and AddOns first. Which options you check during installation depends on which official plugins you're using;
      • If you're using Adamantium Armor, check Adamantium Pauldrons Enchant Fix.
      • If you're using Bitter Coast Sounds, check Better Dragonfly Texture and optionally, Silent BCSounds if you don't want the ambient sounds added by that plugin.
      • If you're using Master Index, check Better Master Index Mesh.
      • If using Siege at Firemoth, check Grurn the Lich Sounds.
      • If using LeFemm Armor, check Sirollus Saccus Training Fix.

Finalizing

Distant Land

First, we'll want to download a groundcover mod, as they function through MGE XE's distant land system. There's two options for groundcover that I can recommend, compare the two and choose one.

  1. Remiros' Groundcover - Includes rocks and mushrooms along with grass. Grasses are lush, but not very tall, making sure they're easy to see through. Grasses have been removed from certain areas of the landscape where they might cause issues or look off. More mod support out-of-the-box than Aesthesia. Download the main Remiros' Groundcover file.
    • When installing, check Core. If you're using Tamriel Rebuilt, check TR Plugins and TR Preview Plugins if you're playing with TR's preview plugin enabled.
    • If you don't like the mushrooms added by this mod, you can check No Mushrooms.
    • For grass that appears more dense, check Thicker Grass.
    • If you're using Solstheim Tomb of the Snow Prince, hide/delete Rem_Solstheim.esp. The Vvardenfell and Solstheim Worldspace Remaster installer includes a patch for Remiros' Groundcover; just run the installer again and check Remiros' Groundcover for S-TOTSP.
  2. Aesthesia Groundcover - Includes ferns, flowers, and other plants along with grass. Grasses are lush, often very tall, and cover the entire landscape. Less compatible out-of-the-box with some mods. Download the main file.
    • After installing, open the mod's files. Move Grass Vanilla 1.esp and Grass Vanilla 2.esp from the Grass Vanilla folder into the mod's main folder. If you AREN'T using Solstheim Tomb of the Snow Prince, you can also do this for Grass Bloodmoon.esp
    • If you're using Tamriel Rebuilt, move all of the files from the Grass TR folder into the mod's main folder. Do the same for the Grass TR preview folder if you're playing with TR's preview plugin enabled.
  • Vurt's Groundcover - Optional, if you want corals to be added underwater in some areas. Don't download the main file, only the Vurts Corals optional file.
  • Distant Mournhold - Allows you to see Mournhold's taller buildings from any part of the city.

If you need to generate new groundcover files (for example, if you want to use Aesthesia's grass with Solstheim Tomb of the Snow Prince, the Aesthesia mod page has a short guide on how to do so towards the bottom.

Now if you look at your plugins, you'll notice that the groundcover mods add... Quite a few .esp files. We don't actually want any of these plugins to be enabled in-game, so you should disable all of them (including Vurts Corals, if you're using it). Don't hide or delete the files, just uncheck them from MO2's Plugins tab.

Once you've done that, launch MGE XE and navigate to the Distant Land tab. Click Distant land generator wizard to begin the process.

  • On the first window that pops up, click Use current load order. Then, find your Groundcover mod .esp files that we disabled in MO2 and check all of them as well. Click continue.
  • On the next few windows, only change the settings if you think you need to; I would for the most part recommend keeping them at their defaults when you first run the program, and fine-tuning them later if you want to make your game look or run slightly better.

Once you've finished generating distant land, you'll be able to modify a lot of options on MGE's Distant Land tab. Much like with distant land settings, I'd recommend mostly leaving these alone at first and seeing how you're game runs with the default settings before adjusting them.

  • Draw distance adjusts how far you're able to see. I recommend a value somewhere between 5 and 15 cells.
  • Dynamic ripples adds natural waves to the water. If you enable them, I recommend setting the height to 40.
  • The Per-pixel lighting shader offers better dynamic lighting, with the expense being an often heavy FPS loss.
    • If you decide to play with per-pixel lighting enabled, you'll want to download the Negative Lights Remover mod from here.
  • High quality exponential fog and High quality atmosphere & distance coloring will make distant fog appear much more realistic, but can also reduce FPS somewhat.

Note that generated distant land won't update automatically; if you install a new mod that makes any changes to the landscape or structures, you'll need to generate it again.

One last note: if you want to speed up distant land generation a bit in the future, you can download the 4GB Patch and use it on MGEXEgui.exe.

mlox

Since we're done installing mods for now, go ahead and launch mlox from MO2. You'll most likely have messages about missing Better Bodies or Robert's Bodies- you can safely ignore these. Just click Update Load Order and then exit the program.

Generally, it's a good idea to run mlox like this every time you install new mods before you play the game. It won't always have any changes to make, but it's best to check anyway.

Wrye Mash

Next, we'll launch Wrye Mash and go through several of its features.

  • If you've launched the game at any point during this guide, you may have gotten an error on startup reading "One or more plugins could not find the correct versions of the master files they depend on." This is usually due to mods being built on versions of the official master files that are dated differently than the ones you have installed, and it can be easily corrected with Wrye Mash.
    • If you look at your mod list on Wrye Mash's Mods tab, you'll notice that most mods have a green check box next to them while some have a yellow or orange one.
    • Click on a mod with an orange or yellow check box, and then click any of the master files listed in the lower right. Wrye Mash will ask if you want to edit/update the masters list. Click Yes, and then click the Save button below the masters list.
    • Repeat this process for any offending mods until all of the boxes are green.
  • Next, we'll create a Multipatch. This is similar to the Bashed Patch for newer Bethesda games.
    • Click Misc at the top of the program, then TES3cmd > Create MultiPatch.
    • The created patch won't have much in it right now, since we've mostly installed mods that only affect graphics. Like mlox, though, it's a good idea to remake your multipatch everytime you install new mods to prevent possible conflicts.
